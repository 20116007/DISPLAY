# display-format — Specification Document

A TypeScript library that formats numeric values for human-readable display in UI applications. It returns both a **display string** (for the screen) and a **calculation-safe numeric value** (for further math). Designed for use in Electron, React, and Node.js applications alongside `unit-convert`.

## Overview

| Property | Value |
|---|---|
| Package name | `display-format` |
| Version | `1.0.0` |
| Module type | CommonJS |
| Entry point | `dist/index.js` |
| Type definitions | `dist/index.d.ts` |

**Purpose:** Convert raw `number` values into:
- `display` — formatted string shown in the UI
- `value` — numeric value safe to use in further calculations (may be rounded/truncated per rule)

**Single public function:** `displayFormat(value, options?)`

## Return Value Design

Unlike a simple `string` formatter, `displayFormat` returns an object with two fields:

```typescript
interface DisplayFormatResult {
  value: number;    // calculation-safe numeric value
  display: string;  // UI formatted string
}
```

| Field | Purpose |
|---|---|
| `display` | What the user sees in an input field or label (e.g. `"56.00000"`, `"1.000E-05"`). |
| `value` | What your app should store or pass to the next calculation. May be rounded or truncated to match the display precision. |

### Behavior by rule type

| Rule | `display` | `value` |
|---|---|---|
| `fixed` | `value.toFixed(decimals)` | Original input number (unchanged) |
| `integer` | `Math.round(value)` as string | Rounded integer |
| `scientific` | Scientific notation string | Truncated mantissa × 10^exponent (derived numerically, not parsed from string) |
| `step: 1` | Rounded integer string | Rounded integer |
| `null` / `undefined` / `NaN` / `Infinity` | `""` | `0` |

## Formatting Rules

`getFormatRule(value)` uses the **absolute value** to select a rule:

| Absolute Value Range | Rule Type | Decimal Places |
|---|---|---|
| `< 0.1` | scientific | 3 |
| `< 10` | fixed | 6 |
| `< 100` | fixed | 5 |
| `< 1,000` | fixed | 4 |
| `< 10,000` | fixed | 3 |
| `< 100,000` | fixed | 2 |
| `< 1,000,000` | fixed | 1 |
| `< 100,000,000` | integer | — (rounded) |
| `≥ 100,000,000` | scientific | 3 |

### Processing Flow

```
displayFormat(value, options)
  │
  ├─ null / undefined  → { value: 0, display: "" }
  ├─ NaN / ±Infinity   → { value: 0, display: "" }
  ├─ options.step === 1  → { value: round(value), display: "..." }
  │
  └─ getFormatRule(value)
       ├─ "fixed"       → { value: original, display: toFixed(decimals) }
       ├─ "integer"     → { value: round(value), display: "..." }
       └─ "scientific"  → { value: truncated numeric, display: formatScientific(...) }
```

## Scientific Notation Format

Output pattern: `{sign}{mantissa}E{expSign}{exponent}`

| Component | Format | Example |
|---|---|---|
| Sign | `-` or empty | `-` |
| Mantissa | Fixed decimal, truncated (not rounded) | `1.000` |
| Exponent sign | `+` or `-` | `+` |
| Exponent | 2-digit zero-padded | `05` |

**Examples:**

| Input | `display` | `value` |
|---|---|---|
| `0.00001` | `1.000E-05` | `0.00001` (truncated to 3 mantissa digits) |
| `999999999` | `9.999E+08` | `999900000` |
| `0` | `0.000E+00` | See [Edge Cases](#edge-cases) |

**Truncation:** The mantissa uses `Math.floor` (truncate toward zero). If truncation pushes mantissa to `≥ 10`, the exponent is incremented and mantissa is divided by 10.

For scientific results, `value` is computed from the truncated mantissa and exponent — **not** by parsing the display string back to a number.

```typescript
function displayFormat(
  value: number | null | undefined,
```

```typescript
interface DisplayFormatOptions {
```

```typescript
interface DisplayFormatResult {
  value: number;    // calculation-safe value

}
```

## Installation in Your Electron App

### Local file dependency

```json
{
  "dependencies": {
    "display-format": "file:../display-format"
  }
}
```

```bash
cd display-format && npm run build
cd ../your-electron-app && npm install
```

### Usage in React renderer

```tsx
import { displayFormat } from "display-format";

function ValueDisplay({ raw }: { raw: number }) {
  const { value, display } = displayFormat(raw);

  return (
    <span title={`stored: ${value}`}>
      {display}
    </span>
  );
}
```

### Usage after unit conversion

```typescript
import { convert } from "unit-convert";
import { displayFormat } from "display-format";

const converted = convert("Elong", "m", "mm", 1.23456789);
const { value, display } = displayFormat(converted);

// Show display in UI, store value for next calculation
console.log(display); // "1234.568"
console.log(value);   // 1.23456789 (fixed rule keeps original)
```

## Usage Examples

### Basic formatting

```typescript
import { displayFormat } from "display-format";

displayFormat(56);
// { value: 56, display: "56.00000" }

displayFormat(345.67);
// { value: 345.67, display: "345.6700" }

displayFormat(0.00001);
// { value: 0.00001, display: "1.000E-05" }

displayFormat(999999999);
// { value: 999900000, display: "9.999E+08" }
```

### Use `display` for UI, `value` for state

```typescript
function handleBlur(rawInput: string) {
  const num = Number(rawInput);
  const { value, display } = displayFormat(num);

  setDisplayText(display);   // what user sees
  setStoredValue(value);     // what app calculates with
}
```

### Integer step mode (`step = 1`)

For input fields with `step={1}`:

```typescript
displayFormat(56.7, { step: 1 });
// { value: 57, display: "57" }

displayFormat(56.4, { step: 1 });
// { value: 56, display: "56" }
```

### Safe handling of empty input

```typescript
const userInput: number | null = getInput();
const { value, display } = displayFormat(userInput);

// display === ""  → show empty field
// value === 0     → safe default for calculations
```

## Edge Cases

| Scenario | Behavior |
|---|---|
| Negative values | Absolute value used for rule selection; sign preserved in output |
| Exactly `0.1` | Fixed format (`0.100000`), not scientific |
| Values just below `0.1` (e.g. `0.099999999`) | Scientific notation |
| `null` / `undefined` / `NaN` / `Infinity` | `{ value: 0, display: "" }` |
| Input `0` | `display` is `0.000E+00`; `value` may be `NaN` due to `log10(0)` in scientific path — treat `0` explicitly in your UI if needed |
| Scientific `value` vs `display` | `value` is truncated to match display precision; may differ slightly from raw input |

## Integration with unit-convert

Recommended pipeline for an Electron unit-converter UI:

```typescript
import { convert, validate } from "unit-convert";
import { displayFormat } from "display-format";

function formatConvertedValue(
  group: string,
  fromUnit: string,
  toUnit: string,
  inputValue: number
): DisplayFormatResult | null {
  const validation = validate(group, fromUnit, inputValue);
  if (!validation.valid) return null;

  const converted = convert(group, fromUnit, toUnit, inputValue);
  return displayFormat(converted);
}

// In React component:
const result = formatConvertedValue(group, unit, newUnit, Number(value));
if (result) {
  setDisplayValue(result.display);  // show in input
  setStoredValue(result.value);     // use in next conversion
}
```

| Step | Package | Function | Use |
|---|---|---|---|
| 1. Validate | `unit-convert` | `validate()` | Check input range |
| 2. Convert | `unit-convert` | `convert()` | Unit conversion |
| 3. Format | `display-format` | `displayFormat()` | `display` → UI, `value` → state |
