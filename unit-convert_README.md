# unit-convert — Specification Document

A TypeScript library for engineering unit conversion, validation, and range checking. It is designed to be consumed by desktop applications (Electron), web frontends (React), or Node.js services.

## Overview

| Property | Value |
|---|---|
| Package name | `unit-convert` |
| Version | `1.0.0` |
| Module type | ES Module (`"type": "module"`) |
| Entry point | `dist/index.js` |
| Type definitions | `dist/index.d.ts` |
| Runtime data | `dist/data/units.json` |

**Purpose:** Convert numeric values between units within the same physical group (e.g. `mm` → `in` under `Elong`), validate inputs against SI ranges, and expose min/max limits in any selected unit.

**Supported unit groups:** 39 groups including Elong, Area, Volume, Mass, Pressure, Stress, Temperature, Energy, and domain-specific groups (YarnStress, Metsuke, Kic, etc.).

## Data Model

### `UnitDefinition`

Each unit has two conversion coefficients:

```typescript
interface UnitDefinition {
  multi: number;  // Multiplier
  plus: number;   // Offset (used for temperature and affine transforms)
}
```

### `UnitGroup`

```typescript
interface UnitGroup {
  baseUnit: string;                        // SI or canonical base unit name
  units: Record<string, UnitDefinition>;   // All units in this group
  min_SI?: number;                         // Optional minimum in SI/base unit
  max_SI?: number;                         // Optional maximum in SI/base unit
}
```

### `UnitsDatabase`

```typescript
interface UnitsDatabase {
  [group: string]: UnitGroup;
}
```

### Example JSON Entry

```json
{
  "Elong": {
    "baseUnit": "m",
    "units": {
      "m":  { "multi": 1,       "plus": 0 },
      "mm": { "multi": 1000,    "plus": 0 },
      "in": { "multi": 39.37008,"plus": 0 }
    },
    "min_SI": 0.01,
    "max_SI": 2000000
  }
}
```

## Conversion Mathematics

All conversions go through the group's **base (SI) unit** as an intermediate step.

### To Base Unit

```
baseValue = value / multi + plus
```

### From Base Unit

```
displayValue = (baseValue - plus) * multi
```

### Direct Conversion (A → B)

```
baseValue = toBase(group, unitA, value)
result    = fromBase(group, unitB, baseValue)
```

### Temperature Special Case

When generating data from Excel, the `plus` offset for the `Temperature` group is stored as an absolute value (`Math.abs(plus)`). This supports affine temperature conversions (e.g. °C, °F, K).

## Public API Reference

All functions are exported from the package root:

```typescript
import {
  convert,
  toBase,
  fromBase,
  getUnitGroups,
  getUnits,
  validate,
  getRange
} from "unit-convert";
```

### `getUnitGroups(): string[]`

Returns all available unit group names.

```typescript
const groups = getUnitGroups();
// ["Elong", "Area", "Volume", "Mass", ...]
```

### `getUnits(group: string): string[]`

Returns all unit names within a group. Throws if the group does not exist.

```typescript
const units = getUnits("Elong");
// ["m", "um", "mm", "cm", "in", "ft", "km"]
```

### `toBase(group, unit, value): number`

Converts a value in the given unit to the group's base (SI) unit.

### `fromBase(group, unit, baseValue): number`

Converts a base (SI) value to the given unit.

### `convert(group, fromUnit, toUnit, value): number`

Converts a value directly between two units in the same group.

```typescript
const inches = convert("Elong", "mm", "in", 25.4);
// 1
```

### `validate(group, unit, value): ValidationResult`

Checks whether a value falls within the group's SI range (if defined).

```typescript
interface ValidationResult {
  valid: boolean;
  valueSI: number;
  minSI: number;
  maxSI: number;
  message?: string;
}
```

- If `min_SI` / `max_SI` are not defined for the group, `valid` is always `true` and `minSI`/`maxSI` are `NaN`.
- If out of range, `message` contains a human-readable error string.

### `getRange(group, unit): UnitRange | null`

Returns the allowed min/max expressed in the selected unit.

```typescript
interface UnitRange {
  min: number;
  max: number;
  unit: string;
}
```

Returns `null` if the group has no SI range defined.

## Unit Database

The database is generated from `UNIT.xlsx` and contains **39 unit groups**. Representative groups:

| Group | Base Unit | Example Units | Has SI Range |
|---|---|---|---|
| Elong | m | m, mm, cm, in, ft, km | Yes |
| Area | m2 | mm2, cm2, m2, in2, ft2 | No |
| Mass | kg | kg, g, lb | — |
| Pressure | Pa | Pa, MPa, psi, bar | — |
| Temperature | K | K, C, F | — |
| Energy | J | J, kJ, cal | — |
| Load | N | N, kN, lbf | — |

To regenerate the database from Excel:

1. Place `UNIT.xlsx` in the `unit-convert/` root directory.
2. Run `npm run generate` (or `npm run build` which includes generation).

Excel columns used:

| Column | Purpose |
|---|---|
| `UnitGroup` | Group name (e.g. `Elong`) |
| `Unit` or `Name` | Unit identifier |
| `Multi` | Multiplier coefficient |
| `Plus` | Offset coefficient |
| `min_SI` | Minimum allowed value in SI (base unit row only) |
| `max_SI` | Maximum allowed value in SI (base unit row only) |

## Build Pipeline

```bash
cd unit-convert
npm install
npm run build
```

Build steps (defined in `package.json`):

1. **`npm run generate`** — Runs `excelToJson.ts` to produce `src/data/units.json` from `UNIT.xlsx`.
2. **`tsc`** — Compiles TypeScript from `src/` to `dist/`.
3. **`copyUnitsToDist.ts`** — Copies `src/data/units.json` to `dist/data/units.json`.

> **Note:** If `UNIT.xlsx` is not present, `generate` will fail. For development you can use the pre-generated `src/data/units.json` and run `tsc && node src/scripts/copyUnitsToDist.ts` manually.

## Installation in Your Electron App

### Option A — Local file dependency (recommended for monorepo)

In your Electron app's `package.json`:

```json
{
  "dependencies": {
    "unit-convert": "file:../unit-convert"
  }
}
```

Then:

```bash
cd unit-convert && npm run build
cd ../your-electron-app && npm install
```

### Option B — npm link

```bash
cd unit-convert && npm run build && npm link
cd ../your-electron-app && npm link unit-convert
```

### Renderer vs Main Process

| Process | Direct import | Notes |
|---|---|---|
| **Renderer** (React/Vue) | ✅ Supported | Simplest approach; used in `ui-demo`. |
| **Main process** | ✅ Supported | Use when conversion must stay off the UI thread. |
| **Preload + IPC** | ✅ Recommended for production | Expose a safe API via `contextBridge` (see `ui-demo/electron/preload.ts`). |

Ensure `unit-convert` is built (`dist/` exists) before your app imports it.

## Usage Examples

### Basic Conversion

```typescript
import { convert } from "unit-convert";

const result = convert("Pressure", "MPa", "psi", 1);
console.log(result); // converted value
```

### Populate a Unit Dropdown

```typescript
import { getUnitGroups, getUnits } from "unit-convert";

const groups = getUnitGroups();
const elongUnits = getUnits("Elong");

// Use in <select> options
```

### Validate User Input

```typescript
import { validate, getRange } from "unit-convert";

const result = validate("Elong", "mm", 5000000);

if (!result.valid) {
  const range = getRange("Elong", "mm");
  console.error(
    `Value must be between ${range?.min} and ${range?.max} ${range?.unit}`
  );
}
```

### Convert on Unit Change (live conversion)

```typescript
import { convert } from "unit-convert";

function onUnitChange(
  group: string,
  oldUnit: string,
  newUnit: string,
  currentValue: number
): number {
  return convert(group, oldUnit, newUnit, currentValue);
}
```

### Combine with display-format

```typescript
import { convert } from "unit-convert";
import { displayFormat } from "display-format";

const raw = convert("Elong", "m", "mm", 1.234567);
const display = displayFormat(raw);
// "1234.567"
```

## Error Handling

All lookup functions throw `Error` for invalid inputs:

| Condition | Error Message |
|---|---|
| Unknown group | `Unknown group: {group}` or `Unknown unit group: {group}` |
| Unknown unit | `Unknown unit: {unit}` |

`validate()` and `getRange()` throw on unknown group/unit but return a soft result (`valid: false` or `null`) for range violations.

Always wrap user-driven calls in try/catch when group/unit names come from free text.

## Integration Patterns

### Pattern 1 — Direct import in React renderer (simplest)

Used by the reference `ui-demo` project. Import functions directly in components.

```typescript
import { convert, validate, getUnitGroups, getUnits } from "unit-convert";
```

### Pattern 2 — IPC bridge via Electron preload (production)

1. Register IPC handlers in `electron/main.ts` that call `unit-convert` functions.
2. Expose a typed API in `electron/preload.ts` via `contextBridge.exposeInMainWorld`.
3. Call `window.units.convert(...)` from the renderer.

This keeps `nodeIntegration: false` and `contextIsolation: true` (secure defaults).

### Pattern 3 — Main-process-only conversion

Run all conversion in the main process and send only formatted strings to the renderer. Useful when unit data should not be bundled in the frontend.
