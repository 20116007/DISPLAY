# display-format — Specification Document

A TypeScript library that formats numeric values for human-readable display. It automatically selects decimal precision or scientific notation based on the magnitude of the value. Designed for use in Electron, React, and Node.js applications alongside `unit-convert`.

## Overview

| Property | Value |
|---|---|
| Package name | `display-format` |
| Version | `1.0.0` |
| Module type | CommonJS |
| Entry point | `dist/index.js` |
| Type definitions | `dist/index.d.ts` |

**Purpose:** Convert raw `number` values into display strings with appropriate precision. Handles very small numbers, very large numbers, and integer rounding based on configurable rules.

**Single public function:** `displayFormat(value, options?)`

| File | Role |
|---|---|
| `src/index.ts` | Exports `displayFormat` and `DisplayFormatOptions`. |
| `src/displayFormat.ts` | Entry function: handles null/NaN/Infinity, applies step override, delegates to rules. |
| `src/rules.ts` | `getFormatRule(value)` — selects `fixed`, `integer`, or `scientific` based on absolute value. |

## Formatting Rules

`getFormatRule(value)` uses the **absolute value** of the input to select a rule:

| Absolute Value Range | Rule Type | Decimal Places |
|---|---|---|
| `< 0.1` | scientific | 3 |
| `< 10` | fixed | 6 |
| `< 100` | fixed | 5 |
| `< 1,000` | fixed | 4 |
| `< 10,000` | fixed | 3 |
| `< 100,000` | fixed | 2 |
| `< 1,000,000` | fixed | 1 |
| `< 100,000,000` | integer | — (rounded) |
| `≥ 100,000,000` | scientific | 3 |

### Processing Flow

```
displayFormat(value, options)
  │
  ├─ null / undefined  → ""
  ├─ NaN / ±Infinity   → ""
  ├─ options.step === 1  → Math.round(value).toString()
  │
  └─ getFormatRule(value)
       ├─ "fixed"       → value.toFixed(decimals)
       ├─ "integer"      → Math.round(value).toString()
       └─ "scientific"  → formatScientific(value, decimals)
```

## Scientific Notation Format

Output pattern: `{sign}{mantissa}E{expSign}{exponent}`

| Component | Format | Example |
|---|---|---|
| Sign | `-` or empty | `-` |
| Mantissa | Fixed decimal, truncated (not rounded) | `1.000` |
| Exponent sign | `+` or `-` | `+` |
| Exponent | 2-digit zero-padded | `05` |

**Examples:**

| Input | Output |
|---|---|
| `0` | `0.000E+00` |
| `0.00001` | `1.000E-05` |
| `999999999` | `9.999E+08` |

**Truncation behavior:** The mantissa uses `Math.floor` (truncate toward zero), not standard rounding. If truncation pushes the mantissa to `≥ 10`, the exponent is incremented and the mantissa is divided by 10.

## Public API Reference

### Import

```typescript
// CommonJS
const { displayFormat } = require("display-format");

// TypeScript / ES Module (with compatible bundler)
import { displayFormat } from "display-format";
import type { DisplayFormatOptions } from "display-format";
```

### `displayFormat(value, options?): string`

```typescript
function displayFormat(
  value: number | null | undefined,
  options?: DisplayFormatOptions
): string;
```

### `DisplayFormatOptions`

```typescript
interface DisplayFormatOptions {
  step?: number;  // When step === 1, value is rounded to nearest integer
}
```

### Return Values

| Input | Return |
|---|---|
| `null` | `""` |
| `undefined` | `""` |
| `NaN` | `""` |
| `Infinity` / `-Infinity` | `""` |
| Valid number | Formatted string per rules above |

## Installation in Your Electron App

### Local file dependency

```json
{
  "dependencies": {
    "display-format": "file:../display-format"
  }
}
```

```bash
cd display-format && npm run build
cd ../your-electron-app && npm install
```

### Usage in React renderer

```tsx
import { displayFormat } from "display-format";

function ValueDisplay({ value }: { value: number }) {
  return <span>{displayFormat(value)}</span>;
}
```

### Usage after unit conversion

```typescript
import { convert } from "unit-convert";
import { displayFormat } from "display-format";

const converted = convert("Elong", "m", "mm", 1.23456789);
const label = `${displayFormat(converted)} mm`;
```

## Usage Examples

### Basic formatting

```typescript
import { displayFormat } from "display-format";

displayFormat(56);           // "56.00000"
displayFormat(345.67);       // "345.6700"
displayFormat(5600);         // "5600.000"
displayFormat(34897654);     // "34897654"
displayFormat(0.00001);      // "1.000E-05"
```

### Integer step mode (for step=1 inputs)

When an input field has `step={1}`, pass `{ step: 1 }` to round to integer:

```typescript
displayFormat(56.7, { step: 1 });  // "57"
displayFormat(56.4, { step: 1 });  // "56"
```

### Safe handling of empty inputs

```typescript
const userInput: number | null = getInput();

// Returns "" for null — safe to render directly
const text = displayFormat(userInput);
```

### In a form field on blur

```typescript
function handleBlur(rawValue: string) {
  const num = Number(rawValue);
  const formatted = displayFormat(num);
  setDisplayValue(formatted);
}
```

## Edge Cases

| Scenario | Behavior |
|---|---|
| Negative values | Absolute value used for rule selection; sign preserved in output |
| Exactly `0.1` | Uses fixed format (`0.100000`), not scientific |
| Values just below `0.1` (e.g. `0.099999999`) | Scientific notation |
| `1000000.1` | Integer rule applies (abs < 100,000,000) → rounds to `1000000` |
| Null from empty input field | Returns `""` — no error thrown |

## Integration with unit-convert

Recommended display pipeline for an Electron unit-converter UI:

```typescript
import { convert, validate } from "unit-convert";
import { displayFormat } from "display-format";

function formatConvertedValue(
  group: string,
  fromUnit: string,
  toUnit: string,
  value: number
): string {
  const validation = validate(group, fromUnit, value);
  if (!validation.valid) return "";

  const converted = convert(group, fromUnit, toUnit, value);
  return displayFormat(converted);
}
```

| Step | Package | Function |
|---|---|---|
| 1. Validate input | `unit-convert` | `validate()` |
| 2. Convert units | `unit-convert` | `convert()` |
| 3. Format for display | `display-format` | `displayFormat()` |
